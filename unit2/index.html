Definition / Theory

HTML5 introduced several form elements and input types that make web forms more powerful, semantic, and user-friendly. These elements handle validation, input selection, and state display natively in the browser.

Core form elements:

  • <input> — the most versatile element. Types include: text, password, email, number, date, color, range, file, url, tel, search, submit, reset, radio, checkbox.
  • <textarea> — multi-line text input with rows and cols attributes.
  • <select> + <option> — dropdown list. Supports <optgroup> for grouping options.
  • <button> — clickable button with type="submit/reset/button".
  • <label> — associates a text label with a form control via for="id", improving accessibility.
  • <fieldset> + <legend> — groups related form controls visually and semantically.
  • <datalist> — provides an autocomplete suggestion list for an <input>.
  • <output> — displays the result of a calculation or user action.
  • <progress> — shows the completion progress of a task (value / max).
  • <meter> — represents a scalar measurement within a known range (e.g. disk usage).

HTML5 also introduced attributes like required, placeholder, autofocus, pattern (regex validation), min/max, and step. These enable client-side validation without JavaScript, making forms more robust and accessible.

Definition / Theory

CSS Specificity is the algorithm browsers use to decide which CSS rule wins when multiple rules target the same element. Each selector has a specificity weight — the higher weight wins. If two rules have equal specificity, the last one declared wins (cascade order).

Specificity Hierarchy (lowest to highest):

  • Type selectors (element, pseudo-element): div, h1, ::before → weight: 0-0-0-1
  • Class selectors (class, attribute, pseudo-class): .btn, [type="text"], :hover → weight: 0-0-1-0
  • ID selectors: #header → weight: 0-1-0-0
  • Inline styles: style="color:red" → weight: 1-0-0-0
  • !important — overrides everything (use sparingly).

The specificity is calculated as a 4-column score: (inline, IDs, classes, elements). For example, div#content .card h2 = 0 IDs × 1 class × 2 elements = (0, 1, 1, 2). The browser compares left to right — the highest non-zero column wins. This ensures predictable styling and helps debug when styles don't apply as expected.

Specificity Pyramid
!important
Inline styles (style="...")
ID selectors (#id)
Class / Attribute / Pseudo-class (.class, [attr], :hover)
Element selectors (div, h1, ::before)

Specificity increases from bottom to top; !important breaks the normal cascade

Code Example
specificity.html 
<style>
  /* Specificity: 0-0-0-1 (element) */
  p { color: blue; }

  /* Specificity: 0-0-1-1 (class + element) */
  .special p { color: green; }

  /* Specificity: 0-1-0-0 (ID) */
  #unique { color: red; }

  /* Specificity: 1-0-0-0 (inline) */
  /* <p style="color:orange"> wins over all above */
</style>

<p>Blue (element selector)</p>
<div class="special">
  <p>Green (.special p overrides p)</p>
</div>
<p id="unique">Red (ID wins)</p>
<p id="unique" style="color:orange">Orange (inline wins!)</p>

Definition / Theory

CSS can be applied to a web page using three distinct styling techniques: Inline CSS, Internal (Embedded) CSS, and External CSS. Each has its own use cases, advantages, and limitations.

1. Inline CSS: Styles are applied directly to an HTML element using the style attribute. Example: <h1 style="color:red">. It has the highest specificity (except !important) but mixes content with presentation, violating separation of concerns. Best for quick testing or one-off overrides.

2. Internal CSS: Styles are defined inside a <style> tag within the <head> section of the same HTML page. It keeps styles in one place on the same page — useful for single-page sites or when external files are not feasible. However, it is not reusable across multiple pages.

3. External CSS: Styles are written in a separate .css file and linked via <link rel="stylesheet" href="style.css">. This is the best practice because it completely separates content from presentation, promotes reusability (one file for many pages), improves maintainability, and enables browser caching. Major websites always use external CSS for scalability.

Modern development often combines all three: external for global styles, internal for page-specific overrides, and inline sparingly for dynamic JavaScript-driven changes.

Definition / Theory

CSS can create 3D effects and animations using only CSS — no JavaScript needed. The key CSS modules involved are transforms, transitions, and keyframe animations.

1. CSS Transforms: The transform property lets you rotate, scale, skew, or translate an element.

  • 2D transforms: translate(x, y), rotate(deg), scale(factor), skew(deg).
  • 3D transforms: rotateX(deg), rotateY(deg), rotateZ(deg), translateZ(px), perspective(px).
  • perspective on the parent gives depth; transform-style: preserve-3d allows children to occupy 3D space.

2. CSS Transitions: transition: property duration easing delay smoothly interpolates between states on hover or class change. E.g., transition: transform 0.6s.

3. CSS Animations (@keyframes): Define multi-step animations with @keyframes name { 0% {…} 100% {…} } and apply via animation: name 2s infinite.

A classic flipping card effect uses rotateY(180deg) on hover + backface-visibility: hidden to hide the reverse side. Moving objects use translateX/Y with transitions. All of this is pure CSS — the browser's rendering engine handles the smooth motion using the GPU.

Definition / Theory

Bootstrap is the world's most popular front-end component library for building responsive, mobile-first websites. Developed by Twitter (now open source), it provides pre-built HTML, CSS, and JavaScript components that speed up development dramatically.

Why Bootstrap is important for modern web design:

  • Responsive Grid System: A flexible 12-column layout using .container > .row > .col-*. Breakpoints at sm, md, lg, xl, xxl let you build layouts that work on every screen size.
  • Pre-styled Components: Navbars, buttons, cards, forms, tables, alerts, modals, carousels, badges, and many more — all accessible and responsive out of the box.
  • Cross-browser Consistency: Bootstrap normalises styles across Chrome, Firefox, Safari, Edge — no more fighting browser differences.
  • Utility Classes: Hundreds of single-purpose classes for spacing (m-2, p-3), typography (text-center, fw-bold), colors (text-primary, bg-danger), display, flexbox, and more.
  • Customizable via Variables (Sass): Modify Bootstrap's design tokens (colors, fonts, spacing) to match any brand without editing the core library.
  • JavaScript Plugins: Ready-made JavaScript for modals, tooltips, popovers, dropdowns, scrollspy, and collapse — without writing a single line of JS.

By using Bootstrap, developers can prototype quickly, launch faster, and maintain consistency across large projects. It eliminates repetitive CSS work and is compatible with all modern frameworks (React, Angular, Vue).

Definition / Theory

The CSS Box Model is the fundamental concept that governs how every HTML element is rendered as a rectangular box. Every box consists of four concentric layers: content → padding → border → margin.

The four layers (inside → out):

  • Content: The actual content (text, image, child elements). Sized by width and height.
  • Padding: Space between content and its border. padding: 10px. Transparent, shows background color. Adds to the element's visible size.
  • Border: A perimeter line around the padding. border: 2px solid black. Can be styled with width, style, and color.
  • Margin: Space outside the border, separating this element from others. margin: 15px. Transparent and collapses with adjacent margins.

Total width = width + left padding + right padding + left border + right border + left margin + right margin.

The box-sizing property changes how the total size is computed. box-sizing: border-box makes width include padding and border — much easier to work with. box-sizing: content-box (default) only includes the content area. Most modern CSS resets apply border-box globally to avoid confusing math.

Definition / Theory

Mobile-first is a design philosophy where you start building the smallest screen layout first (mobile), then progressively enhance it for larger screens (tablet, desktop, wide). Bootstrap fully embraces this approach with its responsive breakpoints.

Bootstrap's mobile-first system:

  • The base styles target xs (extra small) — phones under 576px — without any media query.
  • Breakpoints use min-width queries to add styles for larger screens: sm (≥576px), md (≥768px), lg (≥992px), xl (≥1200px), xxl (≥1400px).
  • Grid classes: col-12 col-sm-6 col-md-4 col-lg-3 means: on mobile, full width; on sm, 2 columns; on md, 3 columns; on lg, 4 columns.
  • Utility classes: d-none d-md-block hides on mobile, shows on md and up. order-first order-lg-2 reorders content per breakpoint.

Why mobile-first? Over 60% of web traffic is mobile. Designing for small screens forces you to prioritize content, load faster (less data), and ensure touch-friendly interactions. Adding complexity for larger screens is easier than trying to squeeze a desktop design into a phone. Bootstrap's mobile-first approach makes this natural: you write the mobile layout by default and add min-width overrides for larger screens.

Mobile-First Breakpoints

Each breakpoint adds enhancements on top of the mobile base — never removes

Definition / Theory

The Bootstrap Grid System is a 12-column responsive layout system built with CSS Flexbox. It is Bootstrap's most powerful feature for creating layouts that automatically adapt to different screen sizes.

Core structure:

  • .container — wraps the grid with responsive max-width and horizontal padding. .container-fluid spans full width.
  • .row — a horizontal group of columns. Rows have negative margins to counter container padding. Rows also create horizontal flex containers.
  • .col-* — columns are direct children of rows. Column classes define width fractions based on the 12-column grid (e.g., col-6 = 50% width).
  • Responsive column classes: col-sm-4, col-md-6, col-lg-3 — each sets width at a specific breakpoint and above.
  • Auto-layout: col (no number) makes each column equal width automatically. col-auto fits content.
  • Offset & Order: offset-md-2 pushes a column right. order-first / order-last reorders visually.

Row + col values should sum to ≤ 12. If columns exceed 12, the extra ones wrap to a new line. This structure makes it trivial to create complex, responsive layouts without writing any custom CSS media queries.

Definition / Theory

Bootstrap provides several image utility classes that make images responsive (they scale with the parent container) and visually styled (rounded corners, borders, alignment) without writing custom CSS.

Key Bootstrap image classes:

  • .img-fluid — applies max-width: 100% and height: auto so the image scales down on small screens without overflowing its container. This is the most important class for responsive design.
  • .rounded — adds border-radius: 0.375rem (soft corners).
  • .rounded-circle — makes the image perfectly circular (useful for avatars/profile pics).
  • .img-thumbnail — gives the image a 1px border with rounded corners, like a photo thumbnail.
  • .float-start / .float-end — floats the image left or right within its container.
  • .d-block / .mx-auto — makes the image a block element and centers it horizontally.
  • .figure / .figure-img / .figure-caption — a styled figure container with caption, similar to HTML5's <figure>.

Using these classes, developers can create polished image layouts with zero custom CSS. They work with <img> tags and also with <picture> and <svg> elements. The combination of .img-fluid with responsive grid columns ensures images look perfect on any device.

Definition / Theory

Bootstrap provides a set of table classes that enhance plain HTML tables with striped rows, borders, hover effects, dark mode, and responsiveness — all through simple CSS class additions.

Key Bootstrap table classes:

  • .table — the base class that adds light padding, horizontal dividers, and proper alignment. Always start with this.
  • .table-striped — adds alternating row colors (zebra stripes) for better readability.
  • .table-bordered — adds borders on all sides of every cell.
  • .table-hover — highlights rows when the user hovers over them.
  • .table-dark — inverts the table colors for a dark theme.
  • .table-sm — compacts the table with half the padding (denser layout).
  • Contextual classes: .table-primary, .table-success, .table-danger, .table-warning, .table-info — color individual rows or cells.
  • .table-responsive — wraps the table in a horizontally scrollable container on small screens (prevents layout breakage).

Using these classes, you can build professional-looking data tables instantly. The .table-responsive wrapper is especially important for mobile layouts — it adds a horizontal scrollbar when the table is too wide for the viewport, keeping the page layout intact.

Definition / Theory

Using <div> tags, CSS classes, and box properties (margin, padding, border, width, height) is the foundation of CSS-based page layout. Before flexbox and grid, developers built entire website layouts using <div> elements styled with float, display, and box model properties.

Key concepts demonstrated in this example:

  • <div> — a block-level container with no semantic meaning, used purely for layout and styling.
  • CSS classes — reusable style definitions applied via the class attribute.
  • CSS Box Properties — width, height, margin (external spacing), padding (internal spacing), border (outline), box-sizing, overflow, background.
  • Layout techniques: float (left/right), display: inline-block, overflow: hidden (clearfix), and text-align.

Each column is a .box with defined width, padding, margin, border, and background color. This pattern is the bedrock of CSS layout and is still widely used in email templates and legacy projects.

An admin dashboard is a control panel that gives administrators an overview of the system with key metrics, data tables, alerts, and navigation. Bootstrap makes building such dashboards fast and consistent.

Core components of an admin dashboard:

  • Navigation Bar (.navbar): Fixed top navigation with brand logo, menu links, and user avatar. navbar-expand-lg makes it responsive.

The dashboard layout typically follows a 2-column structure: a narrow sidebar with navigation and a wide main content area. Bootstrap's grid system (col-md-2 + col-md-10) creates this layout easily. The entire dashboard is responsive by default, adapting to mobile screens.

Definition / Theory

The three types of CSS — Inline, Internal, and External — are the three distinct ways to apply styles to HTML elements. Each serves a different purpose in web development.

1. Inline CSS: CSS is written directly inside an HTML element's style attribute. It only affects that specific element. Purpose: quick one-off overrides, JavaScript-driven changes, and email templates where external files are not supported. Example: <p style="color:red; font-size:16px">. Pros: highest specificity, no separate file. Cons: mixes content with presentation, not reusable, hard to maintain.

2. Internal (Embedded) CSS: CSS is placed inside a <style> tag in the HTML <head>. It applies to the entire page. Purpose: single-page websites, styling a specific page differently from others, or when external file linking is not possible. Example: <style> body { font-family: Arial; } </style>. Pros: keeps page styles together, no extra HTTP request. Cons: not reusable across pages, increases page size.

3. External CSS: CSS is written in a separate .css file and linked via <link rel="stylesheet" href="styles.css">. Purpose: large websites, consistent styling across multiple pages, team collaboration. Pros: full separation of concerns, reusable (one file for entire site), browser caching reduces load time, easy maintenance. Cons: extra HTTP request (mitigated by caching).

For any multi-page website, External CSS is the recommended approach because it follows the DRY principle and is the most maintainable. The cascade and specificity rules apply identically across all three methods.

Definition / Theory

CSS Selectors are patterns used to select the HTML elements you want to style. They are the bridge between your stylesheet and the HTML document. Understanding selectors is fundamental to writing efficient CSS.

Types of CSS Selectors:

  • 1. Universal Selector (*): Selects every element on the page. Used in CSS resets. Example: * { margin: 0; padding: 0; box-sizing: border-box; }
  • 2. Type/Element Selector: Selects all elements of a given tag name. Example: h1 { color: blue; } — targets every <h1>.
  • 3. Class Selector (.): Selects elements with a specific class attribute. Example: .btn { background: green; } — targets <div class="btn">.
  • 4. ID Selector (#): Selects a single element with a specific id. Example: #header { height: 60px; } — targets <div id="header">. IDs must be unique.
  • 5. Attribute Selector: Selects elements based on an attribute or attribute value. Examples: [type="text"], [href^="https"], [class*="btn"].
  • 6. Descendant Selector (space): Selects elements nested inside another. Example: div p selects all <p> inside a <div>.
  • 7. Child Selector (>): Selects direct children only. Example: ul > li selects <li> directly under <ul>, ignoring deeper nested ones.
  • 8. Adjacent Sibling Selector (+): Selects the element immediately after a specified element. Example: h2 + p targets the first <p> after each <h2>.
  • 9. Pseudo-class Selector (:): Selects elements in a specific state. Examples: :hover, :focus, :nth-child(2), :first-child, :not().
  • 10. Pseudo-element Selector (::): Selects a specific part of an element. Examples: ::before, ::after, ::first-line, ::first-letter.

Selectors can be combined to be more specific (e.g., div.card > p:first-child). The more specific the selector, the higher its specificity weight.

Definition / Theory

CSS 3D transforms allow you to manipulate elements in three-dimensional space using the transform property with 3D functions. The browser's rendering engine utilises the GPU for hardware-accelerated 3D transformations.

Key concepts:

  • 3D Coordinate System: X-axis (horizontal), Y-axis (vertical), Z-axis (depth — towards/away from viewer). Positive Z moves the element toward you.
  • perspective: Defines the distance from the viewer to the z=0 plane. Smaller values = more dramatic 3D effect. Applied to the parent element. perspective: 600px;
  • transform-style: preserve-3d: Tells the browser that child elements should be rendered in 3D space (not flattened). Essential for nesting 3D objects.
  • backface-visibility: hidden: Hides the back side of an element when it faces away — critical for flip-card effects.

3D Transform Functions:

  • rotateX(angle): Rotates around the X-axis (top-to-bottom flip). Example: rotateX(45deg).
  • rotateY(angle): Rotates around the Y-axis (left-to-right flip). Example: rotateY(180deg) — classic card flip.
  • rotateZ(angle): Rotates around the Z-axis (same as 2D rotate).
  • translateZ(z): Moves the element along the Z-axis (closer or farther).
  • translate3d(x, y, z): Moves in all three axes at once.
  • scaleZ(s): Scales along the Z-axis (affects depth).
  • scale3d(x, y, z): Scales in all three dimensions.
  • matrix3d(...): Combines all transforms into a 4×4 matrix (advanced).

Common use cases: flipping cards, 3D product showcases, image galleries with depth, animated 3D cubes, and perspective-scrolling effects.
