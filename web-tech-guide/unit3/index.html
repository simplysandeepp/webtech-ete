Definition / Theory

AJAX (Asynchronous JavaScript And XML) is a web development technique that lets a web page exchange data with a server in the background and update parts of the page without reloading the whole page. It is not a programming language — it is a combination of existing technologies.

  • Asynchronous: the browser does not freeze; the user can keep interacting while the request is processed.
  • JavaScript: sends the request and handles the response.
  • XMLHttpRequest / fetch: the object/API that actually talks to the server.
  • Data format: the server can reply with JSON (most common today), XML, HTML or plain text.

How AJAX works (steps):

  • A user event (click/type) triggers JavaScript.
  • JS creates an XMLHttpRequest (or uses fetch()) and sends it to the server.
  • The server processes it and returns a response.
  • A callback receives the data and updates only the needed part of the DOM.

Main benefits are speed, less bandwidth, and a smooth user experience (live search, infinite scroll, form auto-save). The drawback is it depends on JavaScript being enabled and can complicate the browser back button.

Code Example
ajax-demo.html 
<button onclick="loadUser()">Load User</button>
<p id="out">Click to fetch data...</p>

<script>
  function loadUser() {
    // 1. Create the XMLHttpRequest object
    const xhr = new XMLHttpRequest();

    // 2. Configure: METHOD, URL, async = true
    xhr.open('GET', 'https://jsonplaceholder.typicode.com/users/1', true);

    // 3. Callback runs when the response is ready
    xhr.onreadystatechange = function () {
      // readyState 4 = done, status 200 = OK
      if (xhr.readyState === 4 && xhr.status === 200) {
        const user = JSON.parse(xhr.responseText);   // parse JSON
        document.getElementById('out').textContent =
          'Name: ' + user.name + ' | Email: ' + user.email;
      }
    };

    // 4. Send the request to the server
    xhr.send();
  }
</script>

Definition / Theory

jQuery is a fast, lightweight JavaScript library whose motto is "write less, do more." Its main purpose is to simplify common JavaScript tasks such as HTML/DOM manipulation, event handling, animation and AJAX, while hiding browser differences.

Purpose of jQuery:

  • Simplified DOM selection using CSS-style selectors: $('#id'), $('.class').
  • Cross-browser compatibility — one code works on all major browsers.
  • Easy event handling, animations and AJAX with short methods.

Advantages:

  • Less code: one line replaces many lines of plain JS.
  • Chaining: $('#box').css('color','red').slideUp().slideDown();
  • Rich effects: hide(), show(), fadeIn(), slideToggle(), animate().
  • Simple AJAX: $.get(), $.post(), $.ajax().
  • Large plugin ecosystem and easy learning curve.

You include it with one CDN line: <script src="jquery.min.js"></script>. The $ sign is the jQuery alias. Most code runs inside $(document).ready() so it executes after the DOM has loaded.

jQuery wraps verbose DOM APIs into short, chainable, cross-browser methods

Code Example
jquery-demo.html 
<!-- Include jQuery from a CDN -->

<button id="btn">Toggle Box</button>
<div id="box">Hello jQuery!</div>

  $(document).ready(function () {

    // Event handling + chaining in one short line
    $('#btn').click(function () {
      $('#box').css('color', 'crimson').slideToggle();
    });

    // Simple AJAX with $.get (write less, do more)
    $.get('https://jsonplaceholder.typicode.com/todos/1', function (data) {
      console.log('AJAX title:', data.title);
    });
  });
</script>

Definition / Theory

Operators are special symbols that perform operations on values (operands) to produce a result. JavaScript provides several categories of operators.

  • Arithmetic: + - * / % ** ++ -- — math operations. % is modulus (remainder), ** is power.
  • Assignment: = += -= *= /= — assign/update values. x += 5 means x = x + 5.
  • Comparison: == === != !== > < >= <= — return true/false. == checks value only; === checks value AND type.
  • Logical: && (AND), || (OR), ! (NOT) — combine boolean conditions.
  • Bitwise: & | ^ ~ << >> — operate on binary bits.
  • String: + concatenates: 'Web' + 'Tech' → 'WebTech'.
  • Ternary (conditional): condition ? a : b — short if-else.
  • typeof: returns the data type of a value.

The most important exam point is the difference between == (loose, type-coercing) and === (strict, no coercion). For example 5 == '5' is true, but 5 === '5' is false. Operators follow a defined precedence (e.g. * before +), which can be overridden with parentheses.

Code Example
operators.js 
// Arithmetic
let a = 10, b = 3;
console.log(a + b);   // 13
console.log(a % b);   // 1  (remainder)
console.log(a ** 2);  // 100 (power)

// Assignment
let x = 5;
x += 3;               // x = x + 3  -> 8

// Comparison: == vs ===
console.log(5 == '5');   // true  (only value, type coerced)
console.log(5 === '5');  // false (value AND type checked)

// Logical
let age = 20;
console.log(age > 18 && age < 60);  // true (AND)

// Ternary (short if-else)
let result = (age >= 18) ? 'Adult' : 'Minor';
console.log(result);   // "Adult"

// typeof
console.log(typeof "hi"); // "string"

Definition / Theory

The DOM (Document Object Model) represents an HTML page as a tree of objects (nodes). DOM manipulation means using JavaScript to read, change, add or remove these elements so the page updates dynamically.

1. Selecting elements:

  • getElementById('id') — single element by id.
  • getElementsByClassName() / getElementsByTagName() — collections.
  • querySelector() / querySelectorAll() — CSS-style selection (most flexible).

2. Changing content & attributes:

  • innerHTML (HTML), textContent (text only).
  • setAttribute(), element.style.color, classList.add/remove/toggle().

3. Creating & removing nodes:

  • createElement() — make a new element.
  • appendChild() / append() — add it to the tree.
  • removeChild() / remove() — delete an element.

These operations let pages respond to user actions in real time — adding list items, toggling classes, validating forms — making the web interactive and dynamic without reloading.

DOM Tree
  • document (root)
    • <html>
      • <head>
      • <body>
        • <h1>
        • <ul> → <li>

JS selects a node from this tree, then reads/edits/adds/removes it (the page updates instantly)

Code Example
dom-ops.html 
<h3 id="title">Old Title</h3>
<ul id="list"><li>Item 1</li></ul>
<button onclick="update()">Run DOM Operations</button>

<script>
  function update() {
    // 1. SELECT + change content
    document.getElementById('title').textContent = 'New Title';

    // 2. STYLE + class manipulation
    document.querySelector('#title').style.color = '#f72585';

    // 3. CREATE a new element and APPEND it
    const li = document.createElement('li');
    li.textContent = 'Item added by JS';
    document.getElementById('list').appendChild(li);

    // 4. REMOVE the first item
    document.querySelector('#list li').remove();
  }
</script>

Definition / Theory

Functions are reusable blocks of code that perform a task and optionally return a value. They promote reusability and avoid repetition (DRY principle).

  • Function declaration: function add(a,b){ return a+b; }
  • Function expression: const add = function(a,b){...};
  • Arrow function: const add = (a,b) => a+b; — short ES6 syntax.
  • Parameters are inputs; arguments are the actual values passed.

Objects are collections of related data and behaviour stored as key–value pairs. A key is a property; a function value is called a method.

  • Create: const car = { brand:'Tata', year:2024 };
  • Access: dot notation car.brand or bracket car['brand'].
  • Methods: functions inside objects; this refers to the current object.

Together, functions and objects are the foundation of JavaScript — objects model real-world entities (a user, a car) while functions/methods define their behaviour. This supports modular, organised, object-oriented code.

Object Structure

↓ contains
Properties (data)
name: 'Riya', age: 20
↓ and
Methods (behaviour)
greet() { return this.name }
Code Example
functions-objects.js 
// 1. Function declaration
function add(a, b) {
  return a + b;            // returns a value
}
console.log(add(5, 3));    // 8

// 2. Arrow function (ES6)
const square = n => n * n;
console.log(square(4));    // 16

// 3. Object with properties AND a method
const student = {
  name: 'Riya',            // property
  age: 20,                 // property
  greet: function () {     // method
    return 'Hi, I am ' + this.name + ', age ' + this.age;
  }
};

console.log(student.name);     // dot access -> "Riya"
console.log(student['age']);   // bracket access -> 20
console.log(student.greet());  // calls the method

Definition / Theory

An array is an ordered collection of values accessed by a numeric index (starting at 0). An object stores data as key–value pairs. Real applications combine them as an array of objects — for example a list of students or products.

Common array methods (very important for exams):

  • push() / pop(): add / remove at the end.
  • forEach(): loop over each element.
  • map(): transform every element into a new array.
  • filter(): keep elements that pass a test.
  • reduce(): combine all elements into one value (e.g. a total).
  • find() / sort() / length: search, order, and count.

A typical application stores records as an array of objects, then uses these methods to add, list, filter and total data. For example, a "Student Marks" app keeps [{name, marks}, ...] and uses filter() to find pass students and reduce() to compute the average — demonstrating real data handling without a database.

Array of Objects
students[0]
{name:'Riya', marks:85}
students[1]
{name:'Amit', marks:40}
students[2]
{name:'Sana', marks:72}

An array (index 0,1,2…) holding objects → filter()/map()/reduce() process the records

Code Example
students-app.js 
// Array of objects (a mini database)
let students = [
  { name: 'Riya', marks: 85 },
  { name: 'Amit', marks: 40 },
  { name: 'Sana', marks: 72 }
];

// ADD a new record with push()
students.push({ name: 'Dev', marks: 90 });

// FILTER students who passed (marks >= 50)
let passed = students.filter(s => s.marks >= 50);
console.log('Passed:', passed.map(s => s.name)); // ["Riya","Sana","Dev"]

// REDUCE to find the average mark
let total = students.reduce((sum, s) => sum + s.marks, 0);
let average = total / students.length;
console.log('Average:', average);  // 71.75

// FIND the topper with sort()
let topper = [...students].sort((a, b) => b.marks - a.marks)[0];
console.log('Topper:', topper.name); // "Dev"

Definition / Theory

Credit-card validation uses JavaScript to check that payment details are in the correct format before sending them to the server. This is client-side validation — it gives instant feedback, reduces server load and improves user experience (final checks still happen on the server).

Fields and their validation rules:

  • Card number: must be 16 digits; often verified with the Luhn algorithm (mod-10 checksum) to catch typos.
  • Cardholder name: only letters and spaces (regex).
  • Expiry date: format MM/YY and a future date.
  • CVV: 3 digits (or 4 for Amex).

Techniques used: regular expressions (/^\d{16}$/), string methods, and the Luhn algorithm which doubles every second digit from the right, sums them, and checks the total is divisible by 10. Validation runs on the form's submit event with preventDefault() so an invalid form is not sent. This protects against accidental wrong entries and basic fraud at the entry point.

↓

↓

↓ valid?
✓ Accept → send to server
↓ invalid
✗ Show error, block submit
Code Example
card-validate.js 
function validateCard(num, name, expiry, cvv) {
  // 1. Card number: exactly 16 digits
  if (!/^\d{16}$/.test(num)) return 'Card must be 16 digits';

  // 2. Luhn algorithm (mod-10 checksum)
  let sum = 0, alt = false;
  for (let i = num.length - 1; i >= 0; i--) {
    let d = parseInt(num[i]);
    if (alt) { d *= 2; if (d > 9) d -= 9; }
    sum += d; alt = !alt;
  }
  if (sum % 10 !== 0) return 'Invalid card number (Luhn check failed)';

  // 3. Name: letters and spaces only
  if (!/^[A-Za-z ]+$/.test(name)) return 'Invalid cardholder name';

  // 4. Expiry: MM/YY format
  if (!/^(0[1-9]|1[0-2])\/\d{2}$/.test(expiry)) return 'Expiry must be MM/YY';

  // 5. CVV: 3 digits
  if (!/^\d{3}$/.test(cvv)) return 'CVV must be 3 digits';

  return 'OK';   // all checks passed
}

  • Mention the Luhn algorithm for card numbers — it impresses examiners and earns extra marks.
  • Use regex for each field (16 digits, 3-digit CVV, MM/YY) and preventDefault() on submit.
  • Mistake: doing only client-side checks — always note the server must re-validate too.
💡 Card checks: "16 - Name - MM/YY - CVV - Luhn."

Definition / Theory

Form handling is the process of capturing user input from an HTML <form> and processing it with JavaScript. Form validation checks that the input is correct and complete before submission.

Two kinds of validation:

  • Client-side (JavaScript): instant feedback, fast, reduces server load.
  • Server-side: the secure final check (a user can disable JS).

Key steps in form handling:

  • Listen for the submit event and call event.preventDefault() to stop the default reload.
  • Read values with form.elements or document.getElementById().value.
  • Validate each field: not empty, valid email (regex), password length, matching confirm password.
  • Show error messages next to fields, or submit if all valid.

Common checks use regular expressions (email: /^[^@]+@[^@]+\.[^@]+$/), the .trim() method for empty checks, and comparison for matching fields. Good validation makes forms reliable, user-friendly and secure.

↓
Validate: empty? email? password match?
↓
All valid → submit / success
↓ else

Code Example
form-validate.html 
<form id="reg">
  <input id="email" placeholder="Email">
  <input id="pwd" type="password" placeholder="Password">
  <button type="submit">Register</button>
  <p id="err"></p>
</form>

<script>
  document.getElementById('reg').addEventListener('submit', function (e) {
    e.preventDefault();                 // stop the page reload
    const email = document.getElementById('email').value.trim();
    const pwd   = document.getElementById('pwd').value;
    const err   = document.getElementById('err');

    // Validation rules
    if (email === '') { err.textContent = 'Email is required'; return; }
    if (!/^[^@]+@[^@]+\.[^@]+$/.test(email)) {
      err.textContent = 'Invalid email format'; return;
    }
    if (pwd.length < 6) {
      err.textContent = 'Password must be 6+ characters'; return;
    }

    err.style.color = 'green';
    err.textContent = 'Registration successful!';
  });
</script>

  • Cover the standard checks: empty, email regex, password length, confirm-password match.
  • Mistake: relying on client-side only — state that server-side validation is still required for security.
💡 Validate order: "Empty → Format → Length → Match."

Definition / Theory

jQuery effects and animations are built-in methods that add motion and visual transitions to web pages with very little code. They make UIs feel smooth and interactive.

Basic effect methods:

  • hide() / show() / toggle(): hide, reveal or switch visibility.
  • fadeIn() / fadeOut() / fadeToggle(): change opacity gradually.
  • slideUp() / slideDown() / slideToggle(): slide elements up/down.

Custom animation:

  • animate({props}, speed): animate any numeric CSS property (width, height, opacity, margin).
  • Speed can be 'slow', 'fast' or milliseconds (e.g. 1000).
  • A callback runs when the animation finishes — enabling chaining/queues.

Chaining lets you run several effects in sequence on one element: $('#box').slideUp().slideDown().fadeOut();. All effects can take a duration and an optional callback. These methods internally handle timers and cross-browser issues, so a designer can build rich animations without writing complex JavaScript — the essence of jQuery's "write less, do more."

Definition / Theory

AJAX client–server architecture adds an extra layer — the AJAX engine (XMLHttpRequest) — between the user interface and the web server. In a classic model the browser sends a request and waits for a full new page. In AJAX, the browser talks to the server in the background and updates only part of the page.

The flow:

  • Client (UI): a user action triggers JavaScript.
  • AJAX engine: the XMLHttpRequest object sends an asynchronous request — the user keeps working.
  • Server: processes the request (often with a database) and returns JSON/XML.
  • Callback: JavaScript receives the data and refreshes only the relevant DOM section.

Role of XMLHttpRequest (XHR): it is the core object that makes AJAX possible. Its key members:

  • open(method, url, async) — set up the request.
  • send() — send it to the server.
  • onreadystatechange — fires as state changes.
  • readyState (0–4; 4 = done) and status (200 = OK).
  • responseText / responseXML — the returned data.

This architecture gives faster, partial updates and a desktop-like experience. Modern code often uses fetch(), but XHR remains the classic foundation examiners ask about.

AJAX Architecture (with AJAX engine)
Client — Browser UI (HTML + CSS + JS event)
↓ JS call
AJAX Engine — XMLHttpRequest (async)
⇅ HTTP request / response
Web Server — processes & queries data
↓
Database — returns JSON / XML

The AJAX engine sits between UI and server so only part of the page refreshes (no full reload)

Code Example
xhr-architecture.js 
// The XMLHttpRequest object — heart of AJAX
const xhr = new XMLHttpRequest();

// open(method, url, async=true)
xhr.open('GET', 'https://jsonplaceholder.typicode.com/posts/1', true);

// Fires every time readyState changes (0 -> 4)
xhr.onreadystatechange = function () {
  // readyState 4 = request finished/response ready
  if (xhr.readyState === 4) {
    if (xhr.status === 200) {              // 200 = OK
      const data = JSON.parse(xhr.responseText);
      console.log('Title:', data.title);  // update DOM here
    } else {
      console.log('Error: ' + xhr.status);
    }
  }
};

// send() dispatches the async request to the server
xhr.send();
